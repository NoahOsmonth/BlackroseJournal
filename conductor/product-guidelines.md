# Product Guidelines\n\n## Brand Voice & Tone\n- **Empathetic & Calm:** The primary tone should be soft, reassuring, and prioritizing user comfort, creating a safe space for reflection.\n- **Encouraging:** While calm, the voice should also be supportive and motivating, acknowledging user progress without being overly energetic or jarring.\n- **Balance:** The intersection of these tones should result in a voice that feels like a "wise, supportive friend" – steady, kind, and gently uplifting.\n\n## Visual Design & UI\n- **Strict Adherence to Example Design:** The visual design MUST strictly follow the design patterns, layouts, and aesthetics established in the `example-design` folder.\n- **Fidelity:** Implementation should match the example design files (e.g., HTML/CSS references) with 100% accuracy regarding spacing, typography, colors, and component structure.\n\n## Accessibility\n- **Design-Led Accessibility:** Accessibility features should be implemented in a way that aligns with the visual structures provided in the example design.\n- **Standard Practices:** While adhering to the visual reference, ensure that standard web/mobile accessibility practices (e.g., semantic HTML, proper ARIA labels where applicable) are maintained implicitly within that structure.
